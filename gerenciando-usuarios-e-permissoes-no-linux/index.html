Configurando e acessando o usuário root

Criando a senha para o usuário root

No processo de instalação do Linux, você configura um super-usuário com privilégios de administrador, mas não é a mesma coisa que o usuário root, o usuário root é o administrador do sistema, ele tem permissão para executar qualquer comando. Porém esse usuário por padrão não vem com uma senha definida, o que impossibilita seu acesso. Para criar uma senha para o usuário root, deve se usar o comando sudo passwd root, o prompt vai pedir a senha do seu usuário e depois você deve colocar a senha desejada para o root e confirmá-la.

Entrar como usuário root

Para fazer o login como root, basta usar o comando su, e colocar a senha que você acabou de definir. Para saber qual usuário você está logado pode usar whoami e o prompt retornará o nome do usuário atual.

Gerenciando usuários

Criando um usuário

Para criar um novo usuário, deve-se usar o comando useradd [flags] nome do usuário.

flag Descrição
-m Cria um diretório para o usuário no /home
-c Adiciona um comentário para o usuário, útil para escrever o nome para exibição
-s define a shell padrão do usuário
-p define uma senha para o usuário, porém a senha já deve estar criptografada
-G define os grupos que o usuário fará parte

Fazendo o primeiro usuário

useradd pedro -c 'Pedro Silva' -m -s /bin/bash

O usuário pedro foi criado com um diretório pessoal, mas nesse comando não foi definida uma senha de acesso, para fazer isso, vamos usar o mesmo comando de antes, mas escolhendo o usuário para definir a senha.

passwd pedro

Depois é só colocar a senha desejada.

Outra maneira

Também é possível definir uma senha no momento da criação do usuário adicionando a flag -p, porém é preciso fornecer a senha já cripitografada.

passwd pedro -c 'Pedro Silva -m -s /bin/bash -p $(openssl passwd --password <senha>)

Executando o comandopasswd pedro -e, a senha definida será expirada, assim quando o usuário realizar o login, sera requisitado a criação de uma nova senha.

Modificando o usuário

Caso você precise alterar alguma informação do usuário, ou esqueceu de definir algo durante a criação, pode-se usar o comando usermod, usando a flag que você quer modificar

usermod pedro -s /bin/zsh

Deletando um usuário

Para deletar um usuário, deve-se usar o comando userdel, e caso queira apagar todas as suas informações, adicione a flag -r.

userdel -r pedro

Gerenciando grupos no Linux

Criando um grupo

Para criar um novo grupo deve-se usar o comando groupadd

groupadd FINCANCEIRO

Modificando membros do grupo

Para gerenciar um grupo, deve-se usar o comando gpasswd

Adicionado usuário em algum grupo

gpasswd -a pedro FINANCEIRO

removendo o usuário de um grupo

gpasswd -d pedro FINANCEIRO

Deletando um grupo

groupdel FINANCEIRO

Gerenciando permissões

Como o Linux lida com permissões

O Linux trabalha com três categorias para gerenciar permissões: Dono, grupo e outros, e cada categoria podem ter suas permissões definidas. E exitem três permissões que um arquivo pode ter: Leitura, escrita e execução.

Vendo as informações de permissão em um arquivo

Com o comando ls -l, além de mostrar os arquivos, também é apresentado informações de cada arquivo

Primeira coluna:

A primeira coluna mostra o tipo de arquivo e as permissões de cada grupo

Exemplos

-wrxw-x---

Nesse 1° exemplo é possível saber que se trata de um arquivo que o dono do arquivo tem todas as permissões sobre, o grupo tem permissão de leitura e execução e os outros não tem nenhuma permissão.

drwxrwxrwx

No 2° exemplo é possível saber que se trata de um diretório onde todos os grupos de usuários tem todas as permissões

Dissecando os exemplos

A o primeiro caractere diz o tipo de arquivo

Caractere Tipo
- Arquivo
d Diretório
l Link

e os três caracteres seguintes mostram as permissões do dono do arquivo, e os três seguintes mostram as permissões do grupo, e as três últimas mostram as permissões dos outros.

Segunda coluna:

Mostra somente o número de arquivos da pasta.

Terceira e quarta coluna:

Mostra o dono do arquivo, e a quarta mostra o grupo do arquivo.

Quinta coluna

Mostra o tamanho do arquivo.

Sexta coluna

Mostra a ultima data de modificação do arquivo.

Modificando o dono e o grupo de um arquivo

É possível mudar a propriedade de um arquivo através do comando chown.

chown pedro:FINANCEIRO

Modificado as permissões de um arquivo

Para mudar as permissões vamos usar o comando chmod.

chmod 750 <arquivo>

Dissecando o número:

cada digito do número representa a permissão de um grupo

  • 7 » Dono do arquivo
  • 5 » Grupo do arquivo
  • 0 » Outros

Cada permissão tem um valor predefinido, e é possível gerenciar as permissões com a soma desses números.

Permissão Número
Leitura 1
Escrita 2
Execução 4
  • 7 ( 1 + 2 + 4) » Leitura, escrita e execução
  • 5 (1 + 4) » Leitura e execução
  • 0 » Nenhuma permissão

Observações

Usuários tem um grupo próprio

Por isso tem arquivos que mostram pedro:pedro, quando se cria um arquivo novo, o padrão é adicionar o arquivo no grupo do usuário.

Como ver os grupos e os usuários criados

Os usuários e grupos são anotados em um arquivo

  • Usuários » /etc/passwd
  • Grupos » /etc/groups

Para ver esses arquivos de forma rápida basta usar o comando cat

cat /etc/passwd

é possível filtrar o arquivo também quando se está procurando um usuário ou grupo específico:

cat /etc/passwd | grep pedro

cat /etc/group | FINANCEIRO
